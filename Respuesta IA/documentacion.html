Introducción a MySQL y PostgreSQL

1. Diferencias entre SQLite, MySQL y PostgreSQL

SQLite, MySQL y PostgreSQL son sistemas de gestión de bases de datos (DBMS) con diferentes enfoques y capacidades:

Característica SQLite MySQL PostgreSQL
Tipo Base de datos embebida Relacional, cliente-servidor Relacional, cliente-servidor
Escalabilidad Pequeños proyectos, apps locales Mediana a gran escala Gran escala, alto rendimiento
Soporte de ACID Sí Sí (depende del motor de almacenamiento) Sí
Soporte de JSON Básico Intermedio Avanzado
Extensibilidad Limitada Media Alta (soporta funciones en varios lenguajes)
Uso típico Aplicaciones móviles y locales Web y empresas Analítica, big data, sistemas complejos

2. Instalación y Configuración de MySQL y PostgreSQL

Instalación de MySQL

En Linux (Ubuntu/Debian) 

sudo apt update
sudo apt install mysql-server
sudo systemctl start mysql

En Windows

  • Descargar el instalador desde https://dev.mysql.com/downloads/
  • Seguir las instrucciones del asistente

Configuración Básica 

sudo mysql_secure_installation

Instalación de PostgreSQL

En Linux (Ubuntu/Debian) 

sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql

En Windows

  • Descargar el instalador desde https://www.postgresql.org/download/
  • Seguir el asistente de instalación

Configuración Básica 

sudo -u postgres psql
ALTER USER postgres PASSWORD 'nueva_contraseña';

3. Conexión a MySQL/PostgreSQL con mysql-connector y psycopg2

Conectar a MySQL con mysql-connector 

import mysql.connector

conexion = mysql.connector.connect(
    host="localhost",
    user="usuario",
    password="contraseña",
    database="mi_base_de_datos"
)
cursor = conexion.cursor()
cursor.execute("SELECT VERSION()")
print(cursor.fetchone())
conexion.close()

Conectar a PostgreSQL con psycopg2 

import psycopg2

conexion = psycopg2.connect(
    host="localhost",
    user="usuario",
    password="contraseña",
    database="mi_base_de_datos"
)
cursor = conexion.cursor()
cursor.execute("SELECT version();")
print(cursor.fetchone())
conexion.close()

4. Uso de Claves Primarias, Claves Foráneas y Relaciones entre Tablas

Clave Primaria

Una clave primaria es un identificador único de cada registro en una tabla. 

CREATE TABLE usuarios (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL
);

Clave Foránea y Relaciones

Las claves foráneas permiten establecer relaciones entre tablas. 

CREATE TABLE pedidos (
    id SERIAL PRIMARY KEY,
    usuario_id INTEGER REFERENCES usuarios(id),
    producto VARCHAR(100) NOT NULL
);

Este diseño garantiza integridad referencial entre los datos.

Con esto tienes una introducción completa sobre MySQL y PostgreSQL, su instalación, conexión y manejo de relaciones entre tablas.
